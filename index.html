LUMA 2.0 - AI-Powered Emotional Wellness Companion

LUMA 2.0 is built to create a safe, calming digital space for emotional wellbeing. It combines emotional intelligence, conversational AI, and mindful interventions while keeping all sensitive data local-first and private.

Core Features

  • 💬 AI emotional chatbot with tone-aware Gemini responses
  • 🎨 MoodCanvas that turns emotions into abstract AI art
  • 🫁 BreatheSync exercises personalized to stress levels
  • 📊 Mood analytics dashboard with wellbeing insights
  • 📝 Daily check-ins with guided emotional exercises
  • 🎤 Voice interaction powered by Web Speech API

Tech Stack

React + TypeScript Tailwind CSS + Framer Motion Google Gemini API IndexedDB (local-first) Recharts Web Speech API

Problem & Solution

Many wellness apps feel generic and fail to provide emotionally adaptive support while also risking user privacy. LUMA 2.0 solves this by combining real-time emotion-aware AI guidance with local-first architecture, helping users reflect safely without external data storage.

What I Learned

  • Designing emotionally sensitive AI interactions with thoughtful response tone.
  • Balancing calm UI/UX with rich analytics and personalized workflows.
  • Building privacy-first systems using IndexedDB and local processing patterns.

Deep Learning RAG Interview Prep Agent

A Retrieval-Augmented Generation (RAG) system for deep learning interview preparation. Users upload or reference study materials; the app retrieves relevant context, generates interview-style questions, evaluates responses, and explains answers with citations back to the source content.

What it does

  • 📚 Ingests study materials and builds a searchable knowledge base
  • ❓ Generates contextual interview questions from retrieved chunks
  • ✅ Evaluates answers and surfaces grounded feedback
  • 🔗 Provides source-grounded explanations tied to your documents

Pipeline & stack

End-to-end RAG with LangChain and LangGraph for orchestration, ChromaDB for vector storage, structured chunking and retrieval, and evaluation loops. The UI is built with Streamlit; the LLM runs via Groq API with embedding models for semantic search.

Python LangChain LangGraph ChromaDB Streamlit Groq API Embeddings

Smart Internal Job Matcher & Email Notifier with AI

An AI-powered system that matches employees or applicants to internal roles using Sentence-BERT embeddings, then triggers personalized email notifications. It is designed to cut manual screening time while keeping matches explainable and auditable.

Highlights

  • 🎯 Semantic candidate–job matching with SBERT vector similarity
  • ✉️ Automated, personalized email workflows for shortlisted matches
  • 🔄 Duplicate detection and prevention across applications and roles
  • 🗄️ Dual-database querying for flexible HR and application data access

Architecture

FastAPI serves the REST API; PostgreSQL backs persistent data; the front end is built with React and TypeScript. Embedding and matching logic uses Sentence Transformers, with clear separation between matching rules, notification jobs, and the UI.

Python FastAPI PostgreSQL Sentence Transformers (SBERT) React TypeScript REST API
